# 产品思考三步法 - 项目总结

## 项目概述

这是一个专门为产品经理设计的AI思维框架skill，基于三步思考法：

1. **苏格拉底式提问** - 拆解模糊问题
2. **第一性原理** - 挖掘底层逻辑  
3. **奥卡姆剃刀** - 确定最小可行方案

## 核心特性

### 1. 系统化思维流程
- **澄清**：从表面问题到真实问题
- **拆解**：从模糊到具体（事实、约束、因果）
- **简化**：从复杂到简单（保留充分性）
- **决策**：从分析到行动（可执行方案）

### 2. 实用的领域知识
覆盖产品经理日常工作的核心领域：
- 用户研究和需求分析
- MVP定义和功能优先级
- 用户行为和激励机制
- 数据指标和A/B测试
- 竞争分析和差异化

### 3. 可操作的输出模式
提供标准化的输出格式：
- 快速重构（5-10分钟对话）
- 结构化产品分析（深度思考）
- 需求审计（审查现有需求）
- 优先级决策（多方案比较）
- 假设验证（实验设计）

## 设计原则

1. **实用主义** - 产生可执行的决策，而非聪明的分析
2. **证据导向** - 基于事实和数据，而非观点和假设
3. **简化思维** - 奥卡姆剃刀：最简单的充分方案
4. **验证驱动** - 小步快跑，快速验证，持续迭代

## 与参考项目的差异

本项目基于 [decision-clarity-skill](https://github.com/shanezzzz/decision-clarity-skill)，但做了以下产品化适配：

### 保留的核心逻辑
- 三步思考法（澄清→拆解→简化→决策）
- 假设暴露和约束分析
- 反模式识别和避坑

### 产品经理专用增强
- 产品发现和用户研究知识
- MVP思维和功能优先级框架
- 用户行为和激励机制理解
- 产品指标和验证方法
- 竞争分析和差异化策略
- 丰富的产品决策示例

### 中文本地化
- 全部文档中文化
- 符合中国产品经理语境
- 融合中国互联网产品实践

## 使用方法

### 作为Claude Code Skill
1. 将项目放在本地目录
2. 在Claude Code中引用skill路径
3. AI会自动应用此思维框架

### 手动应用
1. 遇到产品问题时，参考SKILL.md
2. 按照四步工作流思考
3. 查阅相关references获取领域知识
4. 使用output-patterns规范输出

## 质量标准

优秀的产品思考输出应该：
✅ 识别真实的决策或问题
✅ 暴露隐藏假设
✅ 将事实与惯性分开
✅ 减少不必要的复杂性
✅ 保持充分性（不天真简化）
✅ 留给产品经理更清晰的下一步
